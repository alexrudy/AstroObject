9. iraftools – IRAF integration facility¶

This module provides an interface for using AstroObject with PyIRAF

9.1. IRAFObject – Intefrace to IrafTools¶

class AstroObject.iraftools.IRAFObject¶
This is an example class which has used :func:`UseIRAFTools`
iin(statename=None, extension='.fits')¶

Returns a filename for a fits file from the given statename which can be used as input for IRAF tasks. This method should be used for files which will not be modified, as modifications will not be captured by the system. For files which are input, but will be modified, use imod().

Parameters:
  • statename – The name of the frame to use for input.
  • extension – The file extension to use.
Returns:

Filename for use with PyRAF
iout(statename=None, extension='.fits', append=None)¶

Returns a filename for a fits file from the given statename which can be used for output for IRAF tasks. The file is not created, but will be read back into this object when idone() is called.

Parameters:
  • statename – The name of the frame to use for output.
  • extension – The file extension to use.
  • append – A string to append to the state names
Returns:

imod(statename=None, newstatename=None, extension='.fits', append=None)¶

Returns a filename for a fits file from the given statename which can be used as input for IRAF tasks which modify a file in-place. The file will be reloaded when idone() is called.

Parameters:
  • statename – The name of the frame to use for input.
  • newstatename – The name of the frame to use fo the output. If None, uses statename
  • extension – The file extension to use.
  • append – A string to be appended tp the new state name.
Returns:

Filename for use with PyRAF
iinat(*statenames, extension='.fits')¶

Returns a filename for an “@”-list. The “@”-list lists fits files for each frame provided. These fitsfiles are created automatically. The “@”-list should not be used for in-place modification, for that, use imodat().

Parameters:
  • statenames – Names of states to be included in the “@”-list.
  • extension – The file extension to use.
Retunrs :

Filename of the “@”-list
ioutat(*statenames, append=None, extension='.fits')¶

Returns a filename for an “@”-list. The “@”-list lists fits files for each frame provided. These fits-files will be output destinations. They will be re-read into the object when idone() is called.

Parameters:
  • statenames – Names of states to be included in the “@”-list.
  • extension – The file extension to use.
  • append – A string to append to the state names

idone()¶

Cleans up the temporary files, reloading any files which need reloading. Should be called after the IRAF command has completed, before attempting to re-use the stage.
