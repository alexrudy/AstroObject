9. iraftools – IRAF integration facility¶

This module provides an interface for using AstroObject with PyIRAF. For documentation of the individual methods provided by this module, see IRAFToolsMixin. The next two sections will provide a brief overview of the use of iraftools.

To use iraftools, use the UseIRAFTools() function.

AstroObject.iraftools.UseIRAFTools(klass)[source]¶

Class wrapper which allows classes to use IRAF tools. Takes a single parameter, the class name which should have IRAF tools, and creates a dummy class which activates IRAF tools during initialization.

Parameters:klass – The class to apply iraftools to.

Example:

from AstroObject.iraftools import UseIRAFTools
from AstroObject.image import ImageStack
ImageStack = UseIRAFTools(ImageStack)

9.1. File types in iraftools¶

There are three types of FITS files provided by iraftools:

in-file

inpfile and inpatfile

in files are FITS files which are used as input to an iraf fucntion. They are created immediately, and will exist untill the iraftools cleanup stage (iraf.done).

out-file

outfile and outatfile

out files are FITS files which will be output targets for iraf. As such, the filename returned does not point to an existing FITS file. Instead, the filename points to a potential file which will be re-read by the parent stack during the cleanup stage (iraf.done).

mod-file

modfile and modatfile

mod files are FITS files which will be modified in-place by iraf. These files are created immediately, just like in files, and are re-loaded automatically during the cleanup stage (iraf.done). To prevent these frames from overwriting their original content, use the append= keyword to append a string to the new frame name. You could also make a copy of the original frame using

Data["newname"] = Data["oldname"]

Since frame labels are required to be unique, this will copy the old frame.

These file types can be generated individually, or as @lists using functions provided by IRAFToolsMixin.

9.2. Example use of iraftools¶

The best way to understand how iraftools works is to check out the walkthrough example. The example goes a few of the commonly used iraftools methods, and explains their basic operation. The example is explained in detail in Walkthrough of a basic iraftools script.

# -*- coding: utf-8 -*-
# This example assumes that the following files exist:
#  - data.fits
#  - bias.fits (IMAGTYP=zero)
#  - dark.fits (IMAGTYP=dark)
#  - flat.fits (IMAGTYP=flat)
from AstroObject.image import ImageStack
from AstroObject.iraftools import UseIRAFTools
ImageStack = UseIRAFTools(ImageStack) # Enable IRAFTools
from pyraf import iraf
from pyraf.iraf import imred, ccdred
Data = ImageStack.fromFile("data.fits")
Data.read("bias.fits")
Data.read("dark.fits")
Data.read("flat.fits")
Data.select("data")
iraf.ccdproc(
    Data.iraf.modatfile("data",append="-reduced"),
    flat = Data.iraf.infile("flat"),
    dark = Data.iraf.infile("dark"),
    zero = Data.iraf.infile("bias"),
    ccdtype = "",
    fixpix = "no",
    overscan = "no",
    trim = "no",
    zerocor = "yes",
    darkcor = "yes",
    flatcor = "yes",
)
Data.iraf.done()
Data.write(frames=["data-reduced"],filename="final.fits",clobber=True)

A full examle program can be seen in An Example iraftools program.

9.3. IRAFToolsMixin – Intefrace to IRAFTools¶

class AstroObject.iraftools.IRAFToolsMixin(*args, **kwargs)[source]¶

This mixin class enables IRAFTools. By default, tools are only enabled in the class.iraf namespace, so to call infile(), use iraf.infile. To use the IRAFTools shortcuts (documented in Shortcuts for IRAFToolsMixin), pass shortcuts=True to any constructor form for each object, or explicitly call iraf.set_instance_methods().

iraf.inpfile(framename=None, extension='.fits')¶

Returns a filename for a fits file from the given framename which can be used as input for IRAF tasks. This method should be used for files which will not be modified, as modifications will not be captured by the system. For files which are input, but will be modified, use imod().

iraf.done()¶

Cleans up the temporary files, reloading any files which need reloading. Should be called after the IRAF command has completed, before attempting to re-use the stage.

9.4. Shortcuts for IRAFToolsMixin¶

The following shortcuts are bound directly to the stack object by the IRAFToolsMixin.set_instance_methods()

  • iraf.infile = iin()
  • iraf.outfile = iout()
  • iraf.modfile = imod()
  • iraf.inatfile = iinat()
  • iraf.outatfile = ioutat()
  • iraf.modatfile = imodat()
