1. AstroObjectBase¶

The Base API was introduced in version 0.2.1 to facilitate the creation and use of basic template classes.

AstroObjectBase provides template objects for the Object-Oriented Modules AstroImage and AstroSpectra. Template classes implement all of the required methods. However, calling a method defined by a template class will usually raise an AbstractError indicating that an Abstract method was called.

class AstroObjectBase.FITSFrame(label, header=None, metadata=None)[source]¶

A single frame of a FITS image. Frames are known as Header Data Units, or HDUs when written to a FITS file. This frame is generic. It does not legitimately implement any functions. Rather, each function implemented is a placeholder, and will generate an AbstractError if called. Several objects inherit from this one to create HDUs which have some semantic meaning. This object requires a label, and can optionally take headers and metadata.

__call__()[source]¶

Should return the data within this frame, usually as a numpy array.

For an example implementation, see :cls:`AstroImage.ImageFrame`.

__hdu__(primary=False)[source]¶

Retruns a Header-Data Unit PyFits object. The abstract case generates empty HDUs, which contain no data. Subclasses should provide a primary keyword argument, and if that keyword is set, the method should return a primaryHDU.

classmethod __read__(HDU, label)[source]¶

classmethod __save__(data, label)[source]¶

A class save method is called when the parent object is trying to save raw data. In this case, the save method will take raw data and attempt to cast it as an object of its own type, returning such an object. If it can’t do that (because the data doesn’t appear to fit, or for pretty much any other reason) it should raise an AbstractError which will be handled by the calling object.

__show__()[source]¶

Should return a plot object for the current frame, after setting up this plot in matplotlib.

In general, this method should display data nicely, but should not do too much work, so as to be flexible for use with other plotting commands. It is best to do the minimal amount of work to simply show the data. The intent is for the user to call Object.show() when they want a quick view of the data in a given frame.
